# Release 0.4.0 — Linux support

- **Date:** 2026-06-07
- **Scope:** Cut the **0.4.0** release after merging Linux support (PR #8). Minor bump (new platform),
  matching how macOS shipped as 0.3.0.

## Steps

- Merged PR #8 (`feat/linux-support-pulseaudio`) into `master` via a merge commit (`--admin`, since
  branch protection requires a review the solo author can't self-approve; `enforce_admins` is off).
- Bumped version 0.3.1 → **0.4.0** in `package.json`, `src-tauri/Cargo.toml`, `src-tauri/tauri.conf.json`;
  synced `Cargo.lock`.
- `CHANGELOG.md` — new `[0.4.0]` section (Added: Linux support; Changed: CI/release now build on Linux).
- Tagged `v0.4.0` and pushed → `release.yml` builds Windows / macOS / Linux artifacts into a **draft**
  GitHub Release for manual review + publish.

## Notes

- Release artifacts now include Linux `.deb` / `.AppImage` / `.rpm` alongside the Windows installer and
  macOS universal `.dmg`. Builds remain unsigned (first-run OS warnings expected).
- Linux runtime is still **unvalidated on real hardware** (developed on Windows; CI only compiles the
  cfg-gated backend) — same caveat macOS shipped with. Needs a Linux user to confirm live capture.
