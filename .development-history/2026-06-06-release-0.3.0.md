# Release 0.3.0

**Goal:** Cut 0.3.0 — the first release with **macOS** support, shipped as an
experimental "alpha" build alongside the stable Windows build.

## What shipped (since v0.2.0)

- **Added:** macOS support (cpal/CoreAudio capture; system audio via a virtual
  device such as BlackHole; native traffic-light window chrome). Gemini as a
  selectable AI provider alongside OpenAI. Settings: Deepgram API-key help link
  - $200 free-credit note.
- **Fixed:** start-transcription page scrolls when content overflows.

## Process

- **macOS = "sososo Alpha":** `productName` overridden to `sososo Alpha` in
  `tauri.macos.conf.json` (RFC 7396 merge, macOS-only) so the macOS `.app`/`.dmg`
  carry the alpha suffix. Windows keeps `sososo`. Bundle identifier unchanged.
- Bumped version → `0.3.0` in `package.json`, `src-tauri/tauri.conf.json`,
  `src-tauri/Cargo.toml`, `src-tauri/Cargo.lock` (synced via `cargo check`).
- Updated `CHANGELOG.md` ([0.3.0] section + compare links).
- Verified locally: `cargo check` ✓ and `bun run build` ✓.
- Commits `chore(macos): … "sososo Alpha"` + `chore(release): 0.3.0`; annotated
  tag `v0.3.0` → the Release workflow (`tauri-action`) built on both
  `windows-latest` and `macos-latest` (universal `--target universal-apple-darwin`).

## Result

- Draft GitHub Release `v0.3.0` ("sososo v0.3.0") created with assets:
  - `sososo.Alpha_0.3.0_universal.dmg` (~15.4 MB) — macOS, Apple Silicon + Intel
  - `sososo.Alpha_universal.app.tar.gz` (~14.8 MB) — macOS app (updater artifact)
  - `sososo_0.3.0_x64-setup.exe` (~4.9 MB) — Windows NSIS
  - `sososo_0.3.0_x64_en-US.msi` (~6.6 MB) — Windows MSI
- Both release jobs succeeded; pending a manual **Publish**.

## Notes

- This was the first real macOS bundling (CI only runs `cargo clippy`, never
  `tauri build`); Info.plist (mic), Entitlements, the universal build, and DMG
  creation all succeeded on `macos-latest`.
- The Tauri bundle filename sanitizes the space in "sososo Alpha" to a dot
  (`sososo.Alpha_…`); the app display name is "sososo Alpha".
- macOS runtime (audio capture, BlackHole, mic permission) is still UNVERIFIED on
  real hardware — hence "Alpha". See the macOS test checklist in the README /
  `2026-06-06-macos-cross-platform-support.md`.
- Builds are unsigned → Windows SmartScreen / macOS Gatekeeper will warn (on
  macOS, right-click → Open). Publish via the Releases page or
  `gh release edit v0.3.0 --draft=false`.
- Minor: `release.yml` still passes `uploadWorkflowArtifacts` (not a valid
  tauri-action input → ignored with a warning); harmless, optional to remove.
