# Release 0.6.0

**Goal:** Ship in-app auto-update — the first updater-enabled release.

## What shipped (since v0.5.0)

- **Added:** in-app auto-update via the Tauri updater plugin — a silent launch
  check plus a manual check in Settings; banner → download with progress →
  restart. Signed update artifacts + `latest.json` produced by the release workflow.

## Process

- Bumped `0.5.0` → `0.6.0` in `package.json`, `src-tauri/tauri.conf.json`,
  `src-tauri/Cargo.toml`, `src-tauri/Cargo.lock` (synced via `cargo check`).
- Updated `CHANGELOG.md` ([0.6.0] section and compare links).
- Verified locally: `cargo check` ✓ and `bun run build` ✓.
- Commit `chore(release): 0.6.0`, annotated tag `v0.6.0` → the Release workflow
  builds **Windows + macOS (universal) + Linux**, signs updater artifacts (signing
  secrets already configured), and uploads a merged `latest.json`.

## Result

- Tag `v0.6.0` → multi-OS Release workflow produces a **draft** GitHub Release
  (`releaseDraft: true`) for manual Publish after review.

## Notes

- First updater release: updating _to_ 0.6.0 is a one-time manual download (0.5.0
  predates the updater); 0.6.0 → later versions update in-app.
- After publishing, verify the release carries `latest.json` + `.sig` assets with
  all three platforms listed and resolvable URLs.
- Multi-agent repo: only the release-scoped files were staged (explicit paths).
